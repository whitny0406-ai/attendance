# Google Sheets 연동 설정 가이드

출퇴근 관리 시스템을 실제로 사용하려면 아래 3단계 작업이 필요합니다.

---

## 1단계 — Google 스프레드시트 생성 및 시트 초기화

### 1-1. 새 스프레드시트 만들기

1. [Google Sheets](https://sheets.google.com)에 접속합니다.
2. **새 스프레드시트**를 만듭니다.
3. 주소창에서 **스프레드시트 ID**를 복사합니다.
   ```
   https://docs.google.com/spreadsheets/d/【여기가 ID】/edit
   ```

### 1-2. Apps Script로 시트 초기화

1. 스프레드시트 메뉴에서 **확장 프로그램 > Apps Script**를 클릭합니다.
2. 에디터에서 `attendance-api.gs` 파일 전체 내용을 붙여넣습니다.
   - 파일 위치: `output/site/apps-script/attendance-api.gs`
3. 상단 14번째 줄의 `SPREADSHEET_ID`를 실제 ID로 교체합니다.
   ```javascript
   const SPREADSHEET_ID = '여기에_실제_ID_입력';
   ```
4. 함수 드롭다운에서 **`initializeSheets`** 를 선택하고 **실행(▶)** 버튼을 클릭합니다.
5. 권한 요청 팝업이 뜨면 **허용**을 클릭합니다.
6. 스프레드시트로 돌아가면 3개 시트(`직원정보`, `출퇴근기록`, `휴가신청`)가 생성됩니다.

---

## 2단계 — Apps Script 웹앱 배포 (쓰기 API 엔드포인트)

### 2-1. 웹앱으로 배포

1. Apps Script 에디터에서 **배포 > 새 배포**를 클릭합니다.
2. **유형 선택**에서 **웹 앱**을 선택합니다.
3. 다음과 같이 설정합니다.

   | 항목 | 설정값 |
   |------|--------|
   | 설명 | 출퇴근 관리 API v1 |
   | 다음 사용자로 실행 | **나 (스프레드시트 소유자)** |
   | 액세스 권한 | **모든 사용자** |

4. **배포** 버튼을 클릭합니다.
5. 표시되는 **웹앱 URL**을 복사합니다.
   ```
   https://script.google.com/macros/s/【XXXXXXXX】/exec
   ```

> **주의**: 코드를 수정하면 반드시 **새 배포**를 생성해야 변경 사항이 반영됩니다.
> 기존 배포를 수정해도 URL이 이미 배포된 버전을 가리킵니다.

---

## 3단계 — Google Sheets API 키 발급 (읽기 전용)

### 3-1. Google Cloud Console에서 API 키 발급

1. [Google Cloud Console](https://console.cloud.google.com)에 접속합니다.
2. 프로젝트를 생성하거나 기존 프로젝트를 선택합니다.
3. **API 및 서비스 > 라이브러리**에서 **Google Sheets API**를 검색하여 **활성화**합니다.
4. **API 및 서비스 > 사용자 인증 정보**로 이동합니다.
5. **사용자 인증 정보 만들기 > API 키**를 클릭합니다.
6. 생성된 API 키를 복사합니다.

### 3-2. 스프레드시트 공개 설정 (API 키 읽기용)

1. 스프레드시트로 돌아가서 **공유** 버튼을 클릭합니다.
2. **링크가 있는 모든 사용자**로 설정하고 **뷰어** 권한을 줍니다.

> **보안 참고**: API 키는 읽기 전용으로만 사용됩니다.
> 쓰기 작업은 Apps Script 웹앱을 통해서만 처리되어 클라이언트에 서비스 계정 키가 노출되지 않습니다.

## 4단계 — 사이트 설정 파일 수정

`output/site/js/config.js` 파일을 열어 아래 3개 값을 입력합니다.

```javascript
const CONFIG = {
  API_KEY:         '【3단계에서 발급한 API 키】',
  SPREADSHEET_ID:  '【1단계에서 복사한 스프레드시트 ID】',
  APPS_SCRIPT_URL: '【2단계에서 복사한 웹앱 URL】',
  // ...
};
```

## 5단계 — 최초 관리자 계정 등록

시트 구조가 완성되었으면 스프레드시트에서 직접 첫 번째 관리자 계정을 추가합니다.

1. `직원정보` 시트를 엽니다.
2. 2번 행에 아래 형식으로 데이터를 입력합니다.

   | 직원ID | 이름 | 연락처 | 소속 | 직종 | 권한(관리자/직원) | 비밀번호(해시) | 등록일 | 상태 | 예정출근시간 | 예정퇴근시간 |
   |--------|------|--------|------|------|-------------------|----------------|--------|------|-------------|-------------|
   | admin01 | 홍길동 | 010-0000-0000 | 관리팀 | 관리자 | 관리자 | (아래 참조) | 2025-01-01 | 재직 | | |

3. **비밀번호(해시)** 컬럼에는 SHA-256 해시값을 입력합니다.
   - 브라우저 콘솔에서 아래 코드를 실행하면 해시값을 얻을 수 있습니다.
     ```javascript
     async function h(t) {
       const b = await crypto.subtle.digest('SHA-256', new TextEncoder().encode(t));
       return Array.from(new Uint8Array(b)).map(x=>x.toString(16).padStart(2,'0')).join('');
     }
     h('원하는비밀번호').then(console.log);
     ```

## 6단계 — 자동 백업 트리거 설정 (선택)

월 1회 자동 백업을 원하면 Apps Script에서 트리거를 등록합니다.

1. Apps Script 에디터에서 왼쪽 **시계 아이콘(트리거)**을 클릭합니다.
2. **트리거 추가**를 클릭합니다.
3. 다음과 같이 설정합니다.

   | 항목 | 설정값 |
   |------|--------|
   | 실행할 함수 | `scheduledBackup` |
   | 이벤트 소스 | 시간 기반 |
   | 시간 기반 트리거 유형 | 월별 |
   | 요일 | 1일 |
   | 시간 | 오전 1~2시 |

## 사이트 배포

### GitHub Pages (무료)

```bash
# output/site 폴더를 GitHub 저장소에 업로드
# Settings > Pages > Source: main 브랜치 / root 또는 /docs 폴더 선택
```

### 로컬 테스트

```bash
# Python 3이 설치된 경우
cd output/site
python -m http.server 8080
# 브라우저에서 http://localhost:8080 접속
```

> **중요**: 파일을 브라우저에서 직접 열면(`file://`) Google Sheets API 요청이 CORS 오류로 실패합니다.
> 반드시 HTTP 서버를 통해 접속하거나 GitHub Pages 등 HTTPS 환경에 배포하세요.

## 문제 해결

| 증상 | 원인 | 해결 방법 |
|------|------|-----------|
| 로그인 안 됨 | API 키 또는 스프레드시트 ID 오류 | config.js 값 확인 |
| 출근 기록이 안 됨 | Apps Script URL 오류 또는 웹앱 미배포 | 웹앱 URL 재확인 |
| CORS 오류 | file:// 프로토콜로 직접 실행 | HTTP 서버 사용 |
| 시트를 찾을 수 없습니다 | initializeSheets 미실행 | 1-2 단계 재실행 |
| 권한 없음 오류 | 웹앱 액세스 설정 오류 | 배포 설정에서 "모든 사용자" 확인 |
